commit message 格式

type: 提交类别（下面基本会覆盖提交代码所需，建议只出现下面7种类别，多了会更乱）
feat：新功能（feature）
fix：修补bug
docs：文档（documentation）
style： 格式（不影响代码运行的变动）
refactor：重构（即不是新增功能，也不是修改bug的代码变动）
test：增加测试
chore：构建过程或辅助工具的变动
scope：影响范围（可不写）
subject：对改变的正常描述，最好以动词开头语（尽量详细，必填）
body：详细描述（个人认为没必要，可不写）
footer：修改不兼容变动以及关闭 Issue时（个人认为没必要，可不写）
