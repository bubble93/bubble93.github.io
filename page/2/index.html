docker安装nginx

菜鸟教程

安装docker

参考docker安装笔记

查询docker nginx镜像

docker search nginx

拉取docker nginx镜像

docker pull nginx:latest

查看已拉取镜像

docker images

运行容器

docker run –name nginx-test -p 8080:80 -d nginx

–name nginx-test：容器名称。
-p 8080:80： 端口进行映射，将本地 8080 端口映射到容器内部的 80 端口。
-d nginx： 设置容器在在后台一直运行。

docker中使用mongodb

菜鸟教程

安装docker

参考docker安装笔记

查询docker mongo镜像

docker search mongo

拉取docker mongo镜像

docker pull mongo:latest

查看已拉取镜像

docker images

创建容器

docker run -itd –name mongo -p 27017:27017 mongo –auth

-p 27017:27017 ：映射容器服务的 27017 端口到宿主机的 27017 端口。外部可以直接通过 宿主机 ip:27017 访问到 mongo 的服务。
–auth：需要密码才能访问容器服务。

运行容器

docker start 容器id

容器id可通过 docker ps -a 查询

opencv4 历险记

环境为 ubuntu 18.04

1.E: Could not get lock /var/lib/dpkg/lock - open (11: Resource temporarily unavailable)
E: Unable to lock the administration directory (/var/lib/dpkg/), is another process using it?

描述：执行sudo apt-get update命令后执行sudo apt-get upgrade报错

解决方案：该问题是由于apt-get命令未执行完毕导致进程被占用，可以使用如下两种方式解决
1.ps -A | grep apt查询出apt命令占用进程，执行sudo kill -9 processnumber杀掉该进程
2. 等待apt命令执行完毕

npm私有组件库

npm install bubble-ui
后再工程中 import BubbleUI from ‘bubble-ui’
提示 These dependencies were not found: bubble-ui
原因是由于打包时未正确设置导出文件名称，与package.json中的main字段不一致导致引入失败

npm publish 发布时提示403
有以下三种可能

  1. npm源并非官方
  2. 未执行npm login
  3. 发布版本号与当前发布版本号相同

linux学习笔记

cal命令报错

执行cal命令提示command not found，检查发现未安装bsdmainutils。
使用apt-get安装bsdmainutils，提示未找到安装包。
执行apt-get update更新apt-get后再次执行apt-get install bsdmainutils
cal命令执行成功

bc命令报错

执行apt-get install bc安装后可以正常执行

docker学习历险记

下载docker一键安装包

  • 官方安装脚本
    curl -fsSL https://get.docker.com | bash -s docker –mirror aliyun
  • 国内安装脚本
    curl -sSL https://get.daocloud.io/docker | sh

运行docker容器

sudo docker run -i -t ubuntu /bin/bash

查看主机名称

hostname

检查容器hosts文件

cat /etc/hosts

检查容器接口

ip a

此处报错：bash: ip: command not found。
因为未安装 iproute2

安装 iproute2

apt-get install -y iproute2

此处报错：未找到该工具

更新 apt-get

apt-get update

再次安装 iproute2

apt-get install -y iproute2

再次检查容器接口

ip a
