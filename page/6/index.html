git:当未完成任务遇到线上bug

Posted on 2019-06-22 | In 版本管理工具 | Comments:

用git碰到了一个有意思的问题
公司git的分支结构是 marster - develop - bubble_dev
我在bubble_dev分支上开发。然后由我领导把我的分支合进 develop 和 master
今天后端突然更新了之前正式系统上的一个接口地址
要合进主分支
但我本地 又进行了一些新功能的开发
如果要把本次修改提交，意味着我现在刚做了一半的新功能也会上去

解决方案:

  • 1
      1. 退回到未开发新功能前的版本
      1. 修改接口地址后再次提交并合并
      1. 然后我再切回当前版本继续开发,并更新远端分支为当前版本
  • 2
      1. 在bubble_dev下再创建bubble_dev_dev分支
      1. bubble_dev作为合并分支使用
      1. 在bubble_dev_dev上进行开发
  • 3
      1. 切换到master分支
      1. 新建bug分支，修改后，合并回master

git暂存stash

Posted on 2019-06-22 | Edited on 2019-09-09 | In 版本管理工具 | Comments:

暂存

git stash save ‘本次暂存的标识名字’
git stash save -u ‘本次暂存的标识名字’
-u 参数可以暂存未提交文件

查看暂存列表

git stash list

恢复暂存 (暂存记录保存在list内,需要通过list索引index取出恢复)

  • 恢复指定的暂存工作（删除暂存）

    git stash pop stash@{index}

  • 恢复指定的暂存工作（不删除暂存）

    git stash apply stash@{index}

删除暂存

  • 删除某个暂存, 暂存记录保存在list内,需要通过list索引index取出恢复

    git stash drop stash@{index}

  • 删除全部暂存

    git stash clear
